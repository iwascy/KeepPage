产品定位：书签优先

若目标是「管理与回访网页链接」，交互应围绕书签，而不是把存档当成默认主路径。存档仍可保留，但是可选增强能力。

存档优先（上一版方向）

  • 单击 → 进入 KeepPage 阅读视图
  • 原网页藏在次要按钮 / 菜单
  • 文案强调「归档 / 阅读 / 导入」
  • 产品心智：离线阅读器

书签优先（本版 v3）

  • 单击 → 打开原网页（主任务）
  • 存档入口降级：卡片上的「书本」按钮 / 菜单项
  • 文案用「书签 / 添加 / 收藏夹」
  • 产品心智：漂亮的书签库；有存档时再提供回看

信息架构建议：列表先保证「找得到、点得开」；存档状态用小图标或角标提示「已保存快照」，需要时再点进去。不要强迫每次点击都进阅读器。
