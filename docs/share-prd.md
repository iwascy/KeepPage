## 一、背景

KeepPage 已具备个人书签保存、列表筛选、多选批量操作、详情与归档预览等能力，但内容默认仅登录用户可见，缺少「把选中的一批书签发给别人」的路径。

典型诉求：

1. 把近期精选的一批链接发给同事 / 朋友。
2. 对方**不需要注册、不需要登录**即可打开。
3. 分享内容应是**用户主动勾选的多条书签**，而不是整个账号或整个文件夹的无差别暴露。
4. 链接即权限：知道链接的人可读，没有更复杂的协作权限体系。

已有 UI Demo（`docs/mockups/share-list-demo.html`）验证了公开列表页的信息架构与视觉方向，可作为访客侧实现基准。

---

## 二、问题定义

当前用户若想把 KeepPage 中的内容外发，只能：

- 逐条复制原站 URL；
- 或截图 / 导出，丢失可点击列表与统一入口。

核心问题不是「再做一个书签列表」，而是：

**如何让用户从已有书签中稳定选出一批内容，生成可外发的只读链接，并在不引入登录门槛的前提下安全地展示给访客。**

## 三、本期定位

### 3.1 本期是什么

V1 的分享是一套 **「多选书签 → 生成公开只读链接 → 访客打开列表」** 的能力：

| 维度 | V1 定义 |
| --- | --- |
| 粒度 | 用户多选的 **N 条普通书签**（不是单条、不是文件夹、不是整库） |
| 可见性 | **仅链接可见**：持有完整分享 URL 的任何人可读，**无需登录** |
| 权限模型 | 公开读 + 所有者写；无协作者、无评论、无密码门 |
| 内容形态 | 只读列表（标题 / 域名 / 备注摘要 / 标签等公开字段） |
| 点击行为 | 默认跳转 **原始网页** |
| 管理能力 | 所有者可创建、查看自己的分享、复制链接、更新条目/文案、撤销 |

### 3.2 本期不是什么

本期**明确不做**：

| 不做 | 说明 |
| --- | --- |
| 单条书签独立分享页 | 可后续用同一基础设施扩展 |
| 按文件夹 / 标签一键分享 | 避免「误暴露整夹」；V1 必须显式多选 |
| 密码保护 / 有效期 / 访问次数上限 | 可进 V1.1 |
| 可编辑协作、评论、点赞 | 非只读协作产品 |
| 访客保存到自己的 KeepPage 账号 | 可进后续增长能力 |
| 私密模式内容的分享 | **永远禁止**（见安全原则） |
| 访客端归档 HTML 在线预览 | V1 默认仅外链；归档预览牵涉公开对象 ACL，单独评估 |
| 搜索引擎收录优化 / SEO 专题页 | 默认 `noindex` |
| 扩展端创建分享 | V1 以 Web 多选为主入口 |

### 3.3 与现有能力的关系

| 能力 | 关系 |
| --- | --- |
| 列表多选（selection mode） | 创建分享的主入口，复用现有选中集合 |
| 批量移动 / 打标 / 删除 | 同级批量动作；分享是新增 batch action |
| 私密模式 | **硬隔离**：私密书签不可进入任何分享 |
| 归档 / 版本 | V1 仅展示「是否有归档」类弱信号（可选），不开放归档文件公开读 |
| API Token / 扩展设备 | 不参与分享鉴权；公开读走独立 public 路由 |

## 四、产品目标

### 4.1 目标

1. 用户在 Web 列表多选 ≥1 条普通书签后，可一键生成公开分享链接。
2. 访客打开链接即可看到只读列表，**全程无需登录**。
3. 链接持有即有读权限；撤销后链接立即失效。
4. 所有者可管理自己创建的分享（改标题/描述、增删条目、排序、复制链接、撤销）。
5. 私密内容、未授权书签、已删除书签不会错误暴露给访客。
6. 公开接口与鉴权接口分离，避免「带用户态接口误开放」。

### 4.2 非目标

- 不做社交关系链（关注、私信）。
- 不做分享内容的全文检索 / 站内搜索（访客页仅本地过滤可做锦上添花，非必须）。
- 不做多用户共同编辑同一分享。
- 不做「分享即同步文件夹」的双向映射。

### 4.3 成功标准（上线后可观测）

| 指标 | 建议目标（上线 2 周内定性观察） |
| --- | --- |
| 创建成功率 | 创建接口成功率 ≥ 99%（排除客户端校验失败） |
| 公开页可用性 | 有效 token 的 2xx 比例 ≥ 99.5% |
| 误暴露事故 | 私密书签出现在公开响应 = **0** |
| 撤销有效性 | 撤销后公开 GET 返回 404/410，无残留正文 |

## 五、设计原则

1. **显式选择才外发。**  
   只有用户勾选进分享的书签才会出现在公开页；文件夹、标签、搜索结果不能隐式整包分享。

2. **最小暴露。**  
   公开响应只返回访客阅读所需字段；不返回用户 id 邮箱、文件夹树、质量报告细节、对象存储 key、内部版本号等。

3. **链接即密钥。**  
   分享 token 必须足够长且不可枚举；不使用自增数字 id 作为公开路径。

4. **私密永不分享。**  
   创建 / 更新时过滤并拒绝私密书签；公开查询链路不得 join 私密表。

5. **撤销优先于复杂 ACL。**  
   V1 不靠密码/过期做精细控制，靠「一键撤销」作为主要风控手段。

6. **只读契约清晰。**  
   访客页不提供登录态操作（收藏、改备注、删除）；文案明确「公开分享 · 只读」。

7. **契约下沉 domain。**  
   分享相关 request/response schema 放在 `packages/domain`，Web/API 共用。

## 六、核心定义

| 名词 | 定义 |
| --- | --- |
| 分享（Share） | 由所有者创建的一份公开只读集合，含标题、描述、条目列表与公开 token |
| 分享条目（Share Item） | 分享中的一条书签引用（含排序位置） |
| 公开 Token | URL 中的不可猜测标识符，持有即有读权限 |
| 所有者 | 创建该分享的 KeepPage 登录用户 |
| 访客 | 打开公开链接的任意用户，通常未登录 |
| 活跃分享 | 状态为 `active`，公开 URL 可访问 |
| 已撤销分享 | 状态为 `revoked`，公开 URL 不可再读出内容 |
| 公开投影（Public Projection） | 从书签实体映射出的、允许对外展示的字段子集 |

## 七、目标用户与场景

### 7.1 目标用户

| 角色 | 诉求 |
| --- | --- |
| 知识整理者 / 设计师 / 开发者 | 把精选阅读清单发给他人 |
| 团队内分享者 | 用一条链接同步「这批值得看」 |
| 访客 | 零摩擦打开列表并点进原文 |

### 7.2 核心场景

**场景 A · 临时精选分享**

1. 用户在首页多选 5～20 条书签。  
2. 点击「分享」→ 填写标题（可选描述）→ 生成链接。  
3. 复制到微信 / 邮件 / IM。  
4. 对方打开即可浏览并点击原文。

**场景 B · 更新已有分享**

1. 用户打开「我的分享」。  
2. 向某分享追加 2 条新书签，或移除过时条目。  
3. 同一链接内容更新，无需再发新链接。

**场景 C · 误发撤回**

1. 用户发现链接发错人或内容敏感。  
2. 在「我的分享」中撤销。  
3. 访客再次打开看到失效页，无法再读列表。

**场景 D · 源书签被删除**

1. 所有者删除了某条已加入分享的书签。  
2. 公开页该条目自动消失（或标记不可用后隐藏）。  
3. 分享其余条目仍可用。

## 九、用户流程

### 9.1 创建分享（所有者 · Web）

```
列表页进入多选
  → 勾选 1..N 条普通书签
  → 批量操作栏点击「分享」
  → 打开「创建分享」对话框
       · 默认标题：可预填「分享的 N 条书签」或首条标题 + 等
       · 可选描述
       · 展示已选条目缩略列表（弹窗内允许取消个别项）
       · 校验：过滤私密/无效 id；若过滤后为 0 则禁止创建
  → 确认创建
  → 成功态：展示完整 URL +「复制链接」+「打开预览」+「完成」
  → 退出多选
```

**入口位置（建议）**

- 主入口：多选工具栏 batch action「分享」。  
- 次入口：设置或侧栏「我的分享」→「新建分享」（新建时仍需走选书流程或空分享后编辑添加）。  
  - V1 建议 **主入口必须做**；「空分享再添加」可只做管理页编辑。

### 9.2 访问分享（访客 · 公开页）

```
打开 /s/:token（或 /share/:token）
  → 若 token 无效 / 已撤销 → 失效页（不泄露是否曾存在的细节差异可统一文案）
  → 若有效 → 拉取公开投影数据
  → 渲染：标题、描述、分享者展示名、条目数、更新时间、书签列表
  → 点击卡片 → 新标签打开 sourceUrl
```

**公开页约束**

- 不出现登录墙、不出现「请登录后查看」。  
- 不出现所有者邮箱；仅展示 **公开昵称**（见 11.3）。  
- 页头可保留 KeepPage 品牌 +「复制链接」+ 公开只读说明。  
- 视觉对齐 Demo 与 `docs/ui-guide`。

### 9.3 管理分享（所有者 · 需登录）

| 动作 | 说明 |
| --- | --- |
| 列表 | 展示我创建的分享：标题、条目数、状态、创建/更新时间、链接预览 |
| 复制链接 | 仅 `active` 可复制 |
| 编辑元信息 | 改 title / description |
| 编辑条目 | 增删书签、调整顺序 |
| 撤销 | 二次确认后 `revoked`；不可恢复为同一 token（可「基于此重新创建」生成新 token，V1 可选） |
| 撤销后列表 | 撤销为终端状态；「我的分享」仍可见已撤销项（可筛选），不提供硬删除（V1） |

### 9.4 源数据变更的联动

| 源变化 | 公开页行为 |
| --- | --- |
| 书签改标题 / 备注 / 标签 | **实时反映**（Live 投影） |
| 书签删除 | 条目从公开列表移除；`itemCount` 下降 |
| 书签进入私密 | 视作不可分享，移除 |
| 所有者注销 | 其下所有分享撤销或级联不可访问 |

> **内容策略（已确认）：Live 投影**，不在创建时冻结全文快照。公开页始终读取当前普通书签的公开字段。

## 十、权限与安全

### 10.1 权限矩阵

| 操作 | 访客（无登录） | 所有者（登录） | 其他登录用户 |
| --- | --- | --- | --- |
| 查看公开列表 | ✅（持有 token） | ✅ | ✅（同访客，仅 token） |
| 创建分享 | ❌ | ✅ | ❌ |
| 编辑 / 撤销 | ❌ | ✅（仅自己的） | ❌ |
| 列举「我的分享」 | ❌ | ✅ | ❌ |
| 通过 token 反查所有者私密库 | ❌ | ❌ | ❌ |

### 10.2 Token 设计

| 项 | 建议 |
| --- | --- |
| 生成 | 密码学安全随机，至少 128 bit 熵（如 22+ 字符 nanoid / 32 hex） |
| URL | `https://{web-host}/s/{token}`（短路径利于传播） |
| 存储 | DB 存 token 明文或 token_hash；若存 hash，则须另存 `token_preview` 供所有者复制——**V1 建议存可逆的 token 列 + 唯一索引**（实现简单；安全依赖熵而非保密 salting） |
| 枚举防护 | 禁止 `/s/1` 类自增；错误响应避免区分「不存在」与「已撤销」的可利用差异（统一 404 文案即可） |

### 10.3 数据暴露白名单（公开 API）

**允许返回：**

- 分享：`title`、`description`、`itemCount`、`updatedAt`、`ownerDisplayName`  
- 条目：`title`、`sourceUrl`、`domain`、`faviconUrl`（或可公开图标 URL）、**`note`（公开）**、`tags[]`（name/color）、`updatedAt`（或相对时间展示）、**`hasArchive`（bool，仅角标，不可预览归档）**

**禁止返回：**

- `userId`、邮箱、API token  
- `folder` / 完整文件夹路径  
- `htmlObjectKey` / 任意对象存储 key / 上传地址  
- 质量分明细、设备信息、私密书签一切字段  
- **真实 `bookmarkId`**（已确认不暴露）；公开列表无需 item 级稳定 id，按 `position` 顺序渲染即可

### 10.4 私密与越权

- 创建 / 更新时：`bookmarkIds` 必须全部属于当前用户且属于**普通书签空间**。  
- 任一 id 不存在或不属于用户 → 整单失败（推荐）或跳过并提示（次选）；**推荐整单失败，避免静默丢数据**。  
- 公开查询：只通过 `share.token` 定位；join 书签时强制 `user_id = share.owner_id` 且非私密表。

### 10.5 滥用与风控（V1 底线）

| 项 | V1 |
| --- | --- |
| 每用户活跃分享数上限 | **50**（已确认） |
| 单分享最大条目数 | **100**（已确认） |
| 创建频率 | 建议同用户 10 次/分钟 软限制 |
| 公开 GET 缓存 | 可短缓存（如 30s）降低打爆；撤销后需能较快失效 |
| robots | `noindex, nofollow` |
| 速率限制 | 公开接口按 IP 限流，防刷库 |

### 10.6 威胁模型（写清边界）

V1 **承诺**：

- 未持有链接的陌生人，不能靠猜 id 列出你的分享内容。  
- 私密模式内容不会进入分享。  
- 撤销后公开读失败。

V1 **不承诺**：

- 链接泄露后的保密（链接等于口令）。  
- 对抗所有者账号被盗后的恶意分享。  
- 对抗服务端被入侵后的数据读取。  
- 分享内容不被接收方二次传播。

产品文案建议在创建弹窗用一行小字说明：  
「任何拿到链接的人都能查看列表中的书签信息，请勿分享敏感内容。」

## 十一、信息架构与 UI

### 11.1 路由

| 端 | 路由 | 鉴权 |
| --- | --- | --- |
| Web 公开 | **`/s/:token` 干净路径**（History + SPA fallback） | 无 |
| Web 管理 | `#/shares`（或等价 hash 路由） | 需登录 |
| Web 创建 | 对话框，无独立路由 | 需登录 |

> 已确认：公开页使用非 hash 路径 `https://{web-host}/s/{token}`，便于 IM 传播与预览。  
> 管理端可继续沿用现有 hash 路由体系（见 `apps/web/src/app/routes.ts`）。部署需保证 nginx/静态托管对 `/s/*` 回退到 SPA。

### 11.2 创建分享对话框（所有者）

字段：

| 字段 | 必填 | 约束 |
| --- | --- | --- |
| 标题 | 是 | 1～80 字，默认生成 |
| 描述 | 否 | 0～500 字 |
| 已选书签 | ≥1 | 展示标题 + 域名，可移除单项 |

主按钮：创建并复制链接。  
成功态：只读 URL 输入框 + 复制 + 预览。

### 11.3 公开列表页（访客）

对齐 Demo，模块：

1. **顶栏**：品牌、复制链接、公开标识。  
2. **分享头图卡**：标题、描述、分享者展示名、条目数、更新时间；（浏览量 V1.1）  
3. **书签区**：网格默认，可选列表切换。  
4. **卡片**：站点色块/favicon、域名、时间、标题、备注摘要、标签、可选「已归档」角标。  
5. **失效页**：统一插画/图标 +「链接无效或已取消分享」。  
6. **页脚**：只读说明 + KeepPage 轻引导（非强制注册弹窗）。

**分享者展示名规则**

- 优先 `users.name`；为空则用邮箱 `@` 前本地部分或「KeepPage 用户」。  
- 不展示完整邮箱。

### 11.4 我的分享（所有者）

列表字段：标题、状态徽章（活跃/已撤销）、条目数、更新时间、操作（复制 / 编辑 / 撤销）。  
空态：引导去列表多选创建。

### 11.5 扩展端

V1 **不要求** extension 改动。  
若多选仅在 Web 存在，则分享 V1 仅 Web + API 即可。

## 十二、数据模型（建议）

### 12.1 `shares`

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| id | uuid PK | 内部 id |
| user_id | uuid FK | 所有者 |
| public_token | varchar unique | 公开路径 token |
| title | varchar(80) | |
| description | text | 可空 |
| status | enum(`active`,`revoked`) | |
| created_at | timestamptz | |
| updated_at | timestamptz | |
| revoked_at | timestamptz | 可空 |

索引：`(user_id, updated_at desc)`、`unique(public_token)`、可选 `(status, public_token)`。

### 12.2 `share_items`

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| share_id | uuid FK | cascade delete |
| bookmark_id | uuid FK | 源书签；源删除时 **CASCADE 删 item** 或 SET 后清理 |
| position | int | 从 0 递增，稳定排序 |
| created_at | timestamptz | |

PK：`(share_id, bookmark_id)` 或单独 uuid + unique(share_id, bookmark_id)。  
索引：`(share_id, position)`。

### 12.3 与私密表

- **禁止** `share_items` 引用私密书签表 id。  
- 校验只打在普通 `bookmarks` 表。

### 12.4 删除语义

| 事件 | 建议 |
| --- | --- |
| 删除书签 | DB 层 FK `ON DELETE CASCADE` 去掉 share_item |
| 撤销分享 | `status=revoked`，保留 items 供所有者回看（可选清空） |
| 删除用户 | cascade 删 shares |

## 十三、API 契约（建议）

所有 schema 落入 `packages/domain`（如 `share.ts`）。

### 13.1 所有者 · 需登录

| 方法 | 路径 | 说明 |
| --- | --- | --- |
| `POST` | `/shares` | 创建 |
| `GET` | `/shares` | 我的分享列表 |
| `GET` | `/shares/:shareId` | 详情（含条目，内部 id） |
| `PATCH` | `/shares/:shareId` | 更新 title/description/items |
| `POST` | `/shares/:shareId/revoke` | 撤销 |
| `POST` | `/shares/:shareId/copy-payload` | 可选：仅返回 URL 字符串 |

**`POST /shares` body 示例**

```json
{
  "title": "设计与产品灵感合集",
  "description": "近期精选",
  "bookmarkIds": ["uuid-1", "uuid-2"]
}
```

**响应要点**

```json
{
  "share": {
    "id": "...",
    "title": "...",
    "description": "...",
    "status": "active",
    "publicToken": "xY7...",
    "publicUrl": "https://app.example/s/xY7...",
    "itemCount": 2,
    "createdAt": "...",
    "updatedAt": "..."
  }
}
```

**`PATCH` body 示例**

```json
{
  "title": "新标题",
  "description": "新描述",
  "bookmarkIds": ["uuid-1", "uuid-3", "uuid-4"]
}
```

> 更新条目时用 **完整有序列表替换**（实现简单、与多选心智一致）。  
> 空数组不允许（至少 1 条）——已撤销除外不可编辑。

### 13.2 访客 · 公开、无登录

| 方法 | 路径 | 说明 |
| --- | --- | --- |
| `GET` | `/public/shares/:token` | 公开投影 |

**响应要点（白名单字段）**

```json
{
  "title": "设计与产品灵感合集",
  "description": "近期精选",
  "ownerDisplayName": "Cyan",
  "itemCount": 8,
  "updatedAt": "2026-07-07T12:00:00.000Z",
  "items": [
    {
      "title": "Linear 的设计系统与信息架构",
      "sourceUrl": "https://linear.app/",
      "domain": "linear.app",
      "faviconUrl": "https://...",
      "note": "极简导航...",
      "tags": [{ "name": "Design System", "color": "#D8D3F4" }],
      "updatedAt": "...",
      "hasArchive": true
    }
  ]
}
```

错误：

- 无效 / 撤销 → `404` + 统一错误码如 `share_not_found`。  
- 不做 `401`（避免暗示需要登录）。

### 13.3 路由装配原则

- 公开路由注册在 **明确的 `/public/*` 前缀**，中间件不挂用户会话强制校验。  
- 所有者路由走现有 session / bearer 鉴权。  
- 禁止复用 `GET /bookmarks` 通过 query 透传「公开」。

---

## 十四、内容与排序规则

1. **排序**：以 `share_items.position` 为准。创建时前端提交 **有序** `bookmarkIds`（多选顺序）；若仅能拿到 set，则按当前列表可见顺序兜底，再不行按 `updatedAt desc`。  
2. **去重**：同一 `bookmarkId` 在同一分享中只能出现一次。  
3. **备注**：公开 `note`；创建弹窗须提示备注会随链接外发。  
4. **标签**：公开 tag 的 name + color；不暴露 tag id。  
5. **空描述**：公开页不展示空描述块。  
6. **失效页**：统一文案「链接无效或已取消分享」；不展示「曾有多少条」；不区分 token 不存在与已撤销。  
7. **浏览次数**：V1 不做；公开页与 API 均不出现 views。

## 十五、边界与异常

| 情况 | 处理 |
| --- | --- |
| 多选含私密书签 | 创建前前端应不可选；后端再拦，返回 400 明确错误 |
| 多选 0 条点分享 | 按钮禁用 |
| 超过最大条目数 | 400 + 提示上限 |
| 超过用户分享数上限 | 400 + 引导撤销旧分享 |
| 编辑时加入他人的 bookmarkId | 400，整单失败 |
| 并发撤销与 GET | GET 见撤销后 404 |
| 公开页图片 / favicon 失败 | 降级默认图标（与主站一致） |
| 所有者改名 | 公开页展示名随后更新 |
