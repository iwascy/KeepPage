### v0.2 修订说明

相较 2026-03-19 的 v0.1，本版本基于用户反馈做了以下重点变更，其余章节语义保持不变：

1. **保存入口以右键菜单为主**：新增浏览器右键"保存到 KP 私密模式"菜单项，Side Panel 的模式切换降级为兜底入口。
2. **Web 端入口收敛到左下角设置**：私密模式不再占用导航栏一级入口，改为"左下角设置 → 私密模式"二级入口，降低存在感。
3. **关窗即退出为默认行为**：Web 侧解锁状态在标签页关闭、主动刷新或结束当前会话后立即锁定，不再依赖长时间保持私密会话。
4. **方案进一步收敛**：本期聚焦 `password-gated private mode`，不做零知识方案、端到端解密或独立云加密库；私密能力通过独立入口、独立查询链路和独立数据表来保证默认隐藏与访问控制。
5. **威胁模型明确**：本期优先解决“默认隐藏 + 二次密码进入 + 普通/私密完全隔离”，不承诺抵御服务端运行时读取、后端高权限访问等更高等级攻击。

## 一、背景

KeepPage 当前已经具备普通网页保存、同步、列表、详情和预览能力，但还没有一套真正独立的“私密内容工作区”。

现有普通链路的问题是：

1. 用户保存敏感页面时，只能混在普通内容里。
2. 首页默认会展示这些内容，不符合“默认隐藏”的预期。
3. 没有一个独立密码入口来控制私密内容的进入。
4. 扩展右键无法直接把内容保存到私密空间。

用户这次的一期目标很明确，不是优先追求密码学级安全，而是优先把产品能力做出来：

1. 右键可以直接保存到 KP 私密模式。
2. 首页默认不展示私密内容。
3. 只有从左下角设置进入私密模式并输入密码，才能看到私密内容。
4. 私密内容和普通内容完全隔离。
5. 关闭网页后默认退出私密模式。

因此，本期 PRD 采用“功能优先”方案：**不做内容加密，不做端到端私密库，只做密码访问控制、前后端隔离、默认隐藏和独立数据链路。**

## 二、本期定位

### 2.1 本期是什么

本期的私密模式是一个“独立空间 + 二次密码进入”的产品能力，目标是：

- 防止私密内容默认出现在普通首页和普通搜索里。
- 防止借用当前设备的人直接看到私密内容。
- 让用户可以清晰地区分“普通保存”和“私密保存”。

### 2.2 本期不是什么

本期**不是**一套强安全方案，不承诺以下能力：

- 不承诺服务端无法看到私密内容。
- 不承诺数据库泄露后私密内容不可读。
- 不承诺零知识或端到端加密。
- 不承诺私密内容在对象存储中以密文保存。

换句话说，本期是“产品层面的私密模式”，不是“密码学层面的加密私密库”。

### 2.3 密码处理原则

- 服务端可以存储私密模式密码的校验信息。
- 服务端**不能**明文保存私密模式密码。
- 建议使用 `Argon2id` 或 `bcrypt` 保存密码哈希。
- 私密内容本身本期允许按现有业务数据方式存储，不做额外内容加密。

## 三、产品目标

### 3.1 目标

1. 支持浏览器右键“保存到 KP 私密模式”。
2. 首页默认不展示任何私密内容。
3. 用户只能通过左下角“设置 -> 私密模式 -> 输入密码”进入私密空间。
4. 私密内容和普通内容在前端列表、接口、数据库表、同步链路中完全隔离。
5. 关闭或刷新 KeepPage 网页后，默认退出私密模式。
6. 私密模式密码在服务端以哈希形式保存，不允许明文保存。

### 3.2 非目标

本期不做：

- 内容加密存储。
- 私密内容端到端解密。
- 私密全文检索增强。
- 多私密库。
- 团队共享私密库。
- 普通内容与私密内容的批量迁移。

## 四、设计原则

1. **功能先落地。**
   先把入口、隔离、密码、退出机制做通，再谈更强安全。
2. **私密内容不混排。**
   私密内容不能出现在普通首页、普通搜索、普通标签和普通文件夹视图中。
3. **密码是二次进入门槛。**
   登录 KeepPage 账号不等于进入私密模式，必须单独输入私密密码。
4. **链路独立。**
   私密内容不能只靠一个 `is_private` 开关附着在普通链路上，必须有独立路由、独立查询和独立存储表。
5. **安全边界写清楚。**
   产品文案必须明确说明：本期私密模式是访问控制和默认隐藏，不是加密保险箱。

## 五、核心定义

| 名词 | 定义 |
| --- | --- |
| 普通模式 | 当前 KeepPage 默认内容空间 |
| 私密模式 | 需要二次输入密码才能进入的独立内容空间 |
| 私密内容 | 保存时明确指定为私密模式的书签、版本、归档和相关元数据 |
| 锁定态 | 用户未输入私密密码时的状态 |
| 解锁态 | 用户已经验证私密密码、当前网页可查看私密内容的状态 |
| 私密会话 | 仅存在于当前 KeepPage 网页内存中的临时进入状态 |

## 六、目标用户与场景

### 6.1 目标用户

- 需要保存后台页面、内部文档、客户资料页面的用户。
- 希望 KeepPage 首页保持干净，不让敏感条目默认露出的用户。
- 在共享电脑或办公环境中使用 KeepPage 的用户。

### 6.2 典型场景

#### 场景 A：右键保存敏感页面

用户在 CRM 或管理后台中浏览详情页，希望通过右键直接保存到私密模式，而不是先进扩展再切模式。

#### 场景 B：首页默认隐藏

用户打开 KeepPage 首页时，只希望看到普通收藏，不希望敏感内容混在最近保存、搜索结果或标签列表中。

#### 场景 C：通过设置进入私密空间

用户需要查看私密内容时，进入左下角设置中的“私密模式”，输入密码后才进入独立私密工作区。

#### 场景 D：关闭网页自动退出

用户关闭 KeepPage 标签页后，私密模式自动失效。下次重新打开页面时，需要再次输入密码。

## 七、方案总览

KeepPage 私密模式 V1 采用以下方案：

### 7.1 入口层

- 扩展右键菜单新增“保存到 KP 私密模式”。
- Web 首页左下角设置中新增“私密模式”入口。

### 7.2 访问控制层

- 每个用户可设置一个私密模式密码。
- 服务端只保存密码哈希，不保存明文。
- 用户输入正确密码后，服务端返回当前网页可用的私密访问会话。

### 7.3 展示层

- 普通首页默认不展示私密内容。
- 私密内容在独立私密工作区中展示。
- 锁定态下不展示私密列表和私密详情。

### 7.4 数据层

- 私密内容与普通内容分表存储。
- 私密内容查询使用独立 API。
- 普通 API 默认永远不返回私密内容。

### 7.5 会话层

- 私密模式进入状态仅存在于当前 KeepPage 网页内存中。
- 刷新页面、关闭页面、退出账号时，私密模式自动失效。

## 八、用户流程

### 8.1 首次启用私密模式

1. 用户进入首页左下角“设置”。
2. 点击“私密模式”。
3. 系统提示设置私密模式密码，并说明当前版本不做内容加密。
4. 用户输入密码并确认。
5. 服务端保存密码哈希。
6. 私密模式启用成功，状态变为“已锁定”。

### 8.2 右键保存到私密模式

1. 用户在网页中点击右键。
2. 选择“保存到 KP 私密模式”。
3. 如果当前浏览器未登录 KeepPage，先引导登录。
4. 如果私密模式尚未启用，引导用户打开侧边栏完成密码设置。
5. 如果私密模式已启用但未解锁，引导用户输入密码。
6. 验证通过后，当前网页按私密模式保存。
7. 内容进入私密链路，不进入普通链路。

### 8.3 首页进入私密模式

1. 用户进入 KeepPage 首页。
2. 默认只看到普通内容。
3. 用户点击左下角“设置”。
4. 点击“私密模式”。
5. 页面显示锁定态说明和密码输入框。
6. 用户输入密码。
7. 验证通过后，进入私密工作区。

### 8.4 退出私密模式

以下任一情况发生时，私密模式默认退出：

- 用户点击“立即退出私密模式”。
- 用户刷新当前 KeepPage 网页。
- 用户关闭当前 KeepPage 标签页。
- 用户退出登录。

## 九、信息架构

### 9.1 Web 侧

- 普通工作区：保持现状，只展示普通内容。
- 左下角设置：新增“私密模式”入口。
- 私密模式锁定页：输入密码、查看状态说明。
- 私密模式工作区：展示私密列表、私密详情和私密预览。

### 9.2 扩展侧

- 右键菜单：
  - `保存到 KeepPage`
  - `保存到 KP 私密模式`
- Side Panel：
  - 保存模式切换：普通 / 私密
  - 私密模式状态：未启用 / 已锁定 / 已解锁

### 9.3 服务端

- 普通链路：继续服务普通书签。
- 私密链路：新增独立密码管理、独立保存、独立列表、独立详情接口。

## 十、功能需求

### 10.1 扩展侧需求

#### 10.1.1 右键菜单

- 在背景脚本中新增“保存到 KP 私密模式”菜单项。
- 点击后直接发起私密保存流程。
- 如果未满足前置条件，优先打开 Side Panel 完成登录或解锁。

#### 10.1.2 Side Panel 私密模式

- 保留普通 / 私密模式切换。
- 新增私密模式密码设置、输入和锁定状态提示。
- 私密模式保存成功后，任务只出现在私密任务视图中。

#### 10.1.3 保存链路隔离

- 私密保存不能进入普通任务队列。
- 私密保存使用独立标识和独立接口。
- 私密内容不会出现在普通任务同步成功回执中。

### 10.2 Web 侧需求

#### 10.2.1 首页默认不展示私密内容

- 普通首页列表不返回私密内容。
- 普通搜索不搜索私密内容。
- 普通标签、文件夹、统计信息不包含私密内容。
- 最近更新、收藏数、侧边栏计数默认排除私密内容。

#### 10.2.2 设置入口

- 复用当前左下角设置区域，新增“私密模式”菜单项。
- 菜单项展示当前状态：
  - 未启用
  - 已锁定
  - 已解锁

#### 10.2.3 私密锁定页

- 展示私密模式说明。
- 展示密码输入框。
- 展示“进入私密模式”按钮。
- 可选展示私密内容数量和最近更新时间，但不展示标题、域名、标签等内容级信息。

#### 10.2.4 私密工作区

- 仅在密码验证通过后进入。
- 展示私密列表、私密详情、私密版本和预览。
- 与普通工作区完全分离，不混排。
- 页面提供“退出私密模式”按钮。

#### 10.2.5 页面关闭默认退出

- 私密会话只保存在当前网页内存。
- 不写入 `localStorage`、`sessionStorage` 持久化登录态。
- 用户刷新或关闭页面后，需要重新输入密码。

### 10.3 服务端需求

#### 10.3.1 私密密码管理

- 提供私密模式启用接口。
- 提供私密密码校验接口。
- 提供私密模式锁定 / 退出接口。
- 服务端只保存密码哈希及必要元数据。

#### 10.3.2 私密内容独立 API

建议新增独立路由：

- `POST /private-mode/setup`
- `POST /private-mode/unlock`
- `POST /private-mode/lock`
- `GET /private/bookmarks`
- `GET /private/bookmarks/:bookmarkId`
- `POST /private/captures/*`

原则是：**普通 `bookmarks`、`captures`、`objects` 路由不直接复用为私密内容接口。**

#### 10.3.3 查询隔离

- 普通查询默认不返回私密内容。
- 私密查询必须要求账号登录 + 私密模式已解锁。
- 服务端需要区分“普通账号会话”和“私密模式会话”。

### 10.4 数据库需求

#### 10.4.1 密码存储

- 私密模式密码不能明文保存。
- 建议字段：
  - `user_id`
  - `password_hash`
  - `password_algo`
  - `password_updated_at`
  - `enabled_at`

#### 10.4.2 私密内容分表

建议新增独立私密表，而不是直接在普通表上补一个布尔字段：

- `private_mode_configs`
- `private_bookmarks`
- `private_bookmark_versions`
- `private_capture_uploads`

#### 10.4.3 内容存储方式

- 本期私密内容允许按业务明文字段存储。
- 但这些字段只能存在私密表中，不能混入普通表。
- 数据库层面不做内容加密。

## 十一、权限与会话设计

### 11.1 双层进入条件

进入私密模式必须同时满足：

1. 用户已登录 KeepPage 账号。
2. 用户已通过私密模式密码校验。

### 11.2 会话范围

- 私密模式会话限定在当前 KeepPage 网页。
- 新开一个 KeepPage 标签页，默认仍然是锁定态。
- 刷新页面后会话失效。

### 11.3 失败与限制

- 密码错误时给出明确提示。
- 可增加连续失败次数限制和短暂冷却。
- 忘记密码时，本期只支持重置私密模式密码，不保证保留既有访问状态。

## 十二、数据隔离要求

### 12.1 前端隔离

- 普通列表和私密列表使用不同数据源。
- 普通详情页不能通过 URL 猜到私密内容。
- 私密页面使用独立路由或独立视图状态。

### 12.2 服务端隔离

- 普通查询服务与私密查询服务分开实现。
- 普通统计接口默认排除私密数据。
- 普通导出、普通搜索、普通 sidebar stats 默认排除私密数据。

### 12.3 数据库隔离

- 普通表不写入私密内容。
- 私密表不参与普通首页查询。
- 后续若要回收或审计，也可按私密表单独处理。

## 十三、UI 与交互建议

### 13.1 文案建议

由于本期不做内容加密，建议在设置页明确提示：

`私密模式用于默认隐藏和密码进入，不等同于加密保险箱。`

### 13.2 首页入口建议

- 左下角设置中新增“私密模式”。
- 若未启用，展示“设置私密密码”。
- 若已启用未解锁，展示“输入密码进入”。
- 若已解锁，展示“进入私密空间”与“退出私密模式”。

### 13.3 右键入口建议

右键菜单建议直接显示两项，而不是二级菜单，降低操作成本：

- `保存到 KeepPage`
- `保存到 KP 私密模式`

## 十四、验收标准

### 14.1 功能验收

1. 用户可以通过右键“保存到 KP 私密模式”保存网页。
2. 首页默认不展示私密内容。
3. 用户可以通过左下角“设置 -> 私密模式 -> 输入密码”进入私密工作区。
4. 私密内容不会出现在普通搜索、普通列表、普通详情和普通统计中。
5. 私密模式关闭或刷新页面后默认退出。
6. 服务端数据库中私密模式密码不是明文。

### 14.2 隔离验收

1. 普通接口不会误返回私密内容。
2. 私密内容只会落到私密表中。
3. 普通任务链路不会误同步私密内容。
4. Web 普通首页和私密工作区可以独立访问、独立退出。

## 十五、风险与权衡

### 15.1 最大风险

由于本期不做内容加密，服务端和数据库管理员理论上仍可访问私密内容，所以这个版本更适合“默认隐藏 + 二次进入”的场景，不适合高安全场景。

### 15.2 产品风险

如果文案不清楚，用户可能误以为这是“真正加密”的保险箱，产生错误预期。

### 15.3 工程风险

如果偷懒复用普通查询，再加一个 `is_private` 字段，后续极易出现：

- 普通列表误展示私密内容
- 普通搜索误搜到私密内容
- 普通统计把私密内容算进去

因此本期仍然必须坚持“独立链路 + 独立表 + 独立接口”。

## 十六、版本规划

### 16.1 V1：功能优先版

范围：

- 右键保存到 KP 私密模式
- 左下角设置进入私密模式
- 私密密码设置与校验
- 私密内容独立保存与独立展示
- 页面关闭默认退出
- 服务端密码哈希保存

### 16.2 V2：安全增强版

未来可选增强：

- 私密内容加密存储
- 本地解密、服务端不可见明文
- 跨设备私密同步
- 更细的私密搜索与恢复能力

## 十七、结论

KeepPage 私密模式 V1 应定义为一套“可用的产品能力”，而不是“强安全能力”。

一期的关键不是把内容做成密文，而是先把这几件事做对：

1. 私密保存入口明确。
2. 首页默认隐藏。
3. 必须通过设置和密码进入。
4. 私密内容与普通内容完全隔离。
